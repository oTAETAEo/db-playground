# Scenario 01: 대용량 결제 내역 조회 및 복합 인덱스 최적화 실험

### 1. 비즈니스 시나리오 및 배경
* **도메인**: 토스 마이데이터 기반 자산 관리 및 결제 플랫폼
* **상황 (유저 시나리오)**:
    * 사용자가 토스 앱의 자산 관리 탭에서 **'지정한 날짜 사이의 카드, 계좌이체 등등 지출 분석'** 기능을 실행하는 상황입니다.
    * 사용자는 전체 내역이 아닌, **[특정 유저(user_id)]가 [특정 기간] 동안 [어떤걸로]로 결제에 [성공(SUCCESS)]한 내역만 최신순으로 필터링**하여 조회합니다.
* **요구사항:** 사용자가 급증하는 상황에서도 마이크로초(ms) 단위의 빠른 응답 속도를 보장하여 사용자 경험을 극대화해야 합니다.
     
## 2. 최적화 대상
특정 유저가 지정한 특정 시작 날짜, 종료 날짜 동안 결제에 성공(`SUCCESS`)한 내역 중, 특정 결제 건만 최신순(`created_at DESC`)으로 화면에 노출합니다.
